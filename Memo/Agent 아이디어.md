
이름 : change-log-history-bot
설명 : changelog 파일로 관리하고, 찾아보기 어려우니 Notion에 테이블로 관리하여 별도 관리 agent
도구 : 
- git fetch
- 현재 브랜치 조회
- 파일 읽기

브랜치 정보 받기.



플로우 : main 에 merge 될 때 트리거 가능하면 좋을 거 같은데, 일단 변경 이력 저장해줘. 요청 -> 해당 브랜치와 main 브랜치와 커밋 이력 조회 -> 변경 파일 조회(맥락 파악) -> 변경 사항 분석 후 내용 정리 -> 노션 page 생성

이름 : error-tracking-bot

설명 : 주기적으로 로그를 확인해서 에러 로그가 떨궈지면 에러 분석, 발생 파일 탐색, 해결 방안 제시(agent 지시문으로 작성), 이슈 생성, hotfix 브랜치 따서 수정 후(테스트 에이전트 사용 후?) PR 날리기.

도구 :

로그 읽기(기간) return error log 파싱

플로우 : 에이전트는 주기적으로 로그/모니터링을 조회 → 에러 발생 감지 → 에러 분석 → 수정 방안(에이전트 지시문) 자동 생성 → 핫픽치 브랜치 생성·수정 → 로컬/컨테이너/CI에서 테스트 실행 → PR 생성 및 이슈 생성 → 알림/사람 승인 → 병합/배포(선택)

시나리오 : 어제 발생한 에러 있어?(user) → error log 파싱 (tool) → 원인 및 발생 지점 분석 (assistant)
